# Contributing to Ye Tracker

Thank you for your interest in contributing to the Ye Tracker app! We welcome contributions of all kinds.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/ye_tracker_app.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Set up the project: `flutter pub get && dart run build_runner build`

## Development Guidelines

### Code Style

- Follow Dart conventions and use `dart format` for formatting
- Run `flutter analyze` to check for lint issues
- We use `flutter_lints` for linting standards
- Use meaningful variable and function names

### Before Submitting a PR

1. Format your code: `dart format .`
2. Run analysis: `flutter analyze`
3. Run tests (if applicable): `flutter test`
4. Ensure your code has no unused imports or variables
5. Write clear commit messages

### Pull Request Process

1. Update the README.md if you've added new features
2. Provide a clear description of the changes
3. Reference any related issues
4. Ensure all tests pass
5. Keep commits atomic and logical

## Reporting Issues

- Check if the issue already exists
- Include device info and Flutter version
- Provide steps to reproduce the issue
- Include error logs if applicable

## Areas for Contribution

- Bug fixes
- Feature improvements
- UI/UX enhancements
- Performance optimizations
- Documentation improvements
- Translation support

## Questions?

Feel free to open an issue with the `question` label or discussion tag if you have questions.
